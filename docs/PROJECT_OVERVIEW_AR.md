# تقرير حالة المشروع - نظام ERP ادارة المصنع

هذا الملف هو نسخة مرجعية سريعة للفريق تجمع ما تم انجازه وما تبقى بشكل واضح.

## 1) نظرة عامة
نقوم ببناء نظام ERP متكامل لادارة عمليات المصنع والمستودع على مستوى تشغيلي يومي، ويغطي:
- الموارد البشرية والموظفين
- الحضور والانصراف (مع جاهزية الربط لاجهزة البصمة)
- الرواتب والمستحقات والخصومات
- المخزون والمنتجات وحركات المواد
- استيراد البيانات من ملفات CSV و XLSX

هدف النظام هو نقل العمل من ملفات متفرقة الى منصة واحدة دقيقة، قابلة للتوسع، وسهلة المتابعة.

## 2) المكدس التقني الحالي
- Frontend: Next.js (App Router) + React + TypeScript + Tailwind + React Query + Zustand
- Backend: NestJS + Prisma
- Database: PostgreSQL
- Auth: JWT + HttpOnly Cookie بسياسات مناسبة للانتاج
- Infra: Redis اختياري حسب البيئة، والنشر عبر Railway او Docker

## 3) نسبة الانجاز التقريبية
- البنية التحتية والربط العام: 90%
- موديول الموظفين والدوام: 95%
- المركز المالي والرواتب: 60%
- المخزون والاستيراد: 10%
- المتوسط العام: حوالي 65%

## 4) ما تم انجازه بنجاح
1. ادارة الموظفين
- اضافة وتعديل بيانات الموظفين وربطها بالاقسام والادوار.

2. اساس الرواتب
- تجهيز قاعدة حساب الراتب الاساسي اعتمادا على اجر الساعة.
- معالجة مشاكل نوع البيانات في الطلبات لضمان ارسال القيم الرقمية بصيغة صحيحة للباك اند.

3. الحضور والانصراف
- تجهيز المسار لربط سجلات الحضور مع اجهزة البصمة.
- معالجة فروقات التوقيت المحلي و UTC.
- حساب التأخير الديناميكي اعتمادا على وقت بداية الدوام مع هامش سماحية.

4. اساس الامان والصلاحيات
- مسارات تسجيل دخول وجلسات مصادقة.
- صلاحيات مبنية على الادوار.
- سجل تدقيق للتغييرات على مستوى النظام.

## 5) المهام المتبقية ذات الاولوية

### A) استكمال المركز المالي
- بناء واجهة السلف وربطها بالموظفين.
- بناء واجهة المكافآت والخصومات الادارية.
- انجاز مسير الرواتب النهائي لتجميع كل المدخلات في معادلة موحدة.

### B) محرك استيراد البيانات
- واجهة رفع ملفات تدعم CSV و XLSX.
- معاينة البيانات قبل الارسال للباك اند.
- مطابقة الاعمدة والتحقق من employeeId قبل الاعتماد النهائي.

### C) نظام المخزون
- شاشة عرض المواد (SKU، الكمية، الوحدة، الحد الادنى).
- نوافذ حركات المخزون (ادخال IN وصرف OUT).
- تنبيهات النقص عند النزول تحت الحد الادنى.

## 6) خطة تنفيذ مقترحة
1. الاسبوع الاول
- اغلاق المركز المالي بالكامل، خاصة مسير الرواتب النهائي.

2. الاسبوع الثاني
- تنفيذ موديول المخزون وحركات المواد.

3. الاسبوع الثالث
- اكمال الاستيراد والتدقيق النهائي والتجاوب على الموبايل.

## 7) قيمة النظام بعد الاكتمال
- توحيد دورة التشغيل اليومية في منصة واحدة.
- تقليل اخطاء الرواتب والحضور والمخزون.
- تسريع استخراج التقارير واتخاذ القرار.
- رفع مستوى الامان والامتثال عبر سجل تغييرات وصلاحيات واضحة.

## 8) خريطة المجلدات المهمة
- الواجهة: warehouse/Factory
- الخدمات الخلفية: work/warehouse-system/backend-nest
- التوثيق والمخططات: work/warehouse-system/docs و work/warehouse-system/schemas

## خلاصة
المشروع وصل مرحلة قوية من البنية الاساسية، والمرحلة التالية تركز على اغلاق المركز المالي ثم المخزون ثم الاستيراد والتلميع النهائي.
